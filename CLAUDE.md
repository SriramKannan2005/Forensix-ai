# ForensiX AI — Claude Code Instructions

## Project
Digital image forensics desktop app + IEEE conference paper.
Root: D:\forensix_ai
Python 3.9, Windows, RTX 4060 (CUDA 12.7), venv at D:\forensix_ai\venv

## Key commands
Activate venv:       venv\Scripts\activate
Run app:             python forensix_ui.py
Run training:        python notebooks\train_cnn.py --data data\cifake,data\casia --epochs 20 --batch 32
Verify datasets:     python -c "from notebooks.train_cnn import load_datasets; ..."

## Folder layout
modules\image\      <- detection modules (ela.py, detector.py, noise_analyzer.py, metadata_extractor.py, cnn_classifier.py)
modules\image\model.pt  <- CNN weights (generated by training, do not create manually)
aggregator\         <- aggregator.py (signal risk profile)
llm\                <- prompts.py, provider.py, reasoning_engine.py
notebooks\          <- train_cnn.py
data\cifake\        <- CIFAKE dataset: train\REAL, train\FAKE, test\REAL, test\FAKE
data\casia\         <- CASIA v2: Au\ and Tp\ subdirs (Gt\ = masks, ignore)
data\faceforensics\ <- FaceForensics++ (optional, later)
outputs\heatmaps\   <- ELA heatmaps saved here
outputs\training_metrics.json <- written by train_cnn.py

## Dataset zip locations (user downloads these)
Downloads folder or wherever user places them.
CIFAKE zip: typically cifake.zip or archive.zip from Kaggle
CASIA zip:  typically CASIA2.0_revised.zip or similar

## Training pipeline
1. Unzip datasets to correct paths
2. Verify folder structure
3. Run: python notebooks\train_cnn.py --data data\cifake,data\casia --epochs 20 --batch 32
4. model.pt auto-saved to modules\image\model.pt on best val_acc

## Signal weights in detector.py
With CNN: CNN 40% + ELA 25% + Noise 20% + Metadata 15%
Without:  ELA 50% + Noise 30% + Metadata 20%

## Do not touch
- modules\image\model.pt (generated, not source)
- outputs\ (runtime generated)
- .env (contains API keys)
